Vídeo que en el que se explica el concepto de herencia a través de una jerarquía de clases relacionada con distintos tipos de trabajadores (llamados funcionarios en el ejemplo). Se define una clase Funcionario con unos miembros básicos para todo trabajador (nombre, dirección, sueldo base, tipo de trabajador, etc.) y a continuación se van definiendo las clases Programador, Analista e Ingeniero, cada uno con  sus métodos y atributos específicos. Después se define la clase Proyecto, con sus atributos y métodos, incluyendo el analista, el programador y el ingeniero asignados a un proyecto. Una vez implementada la clase Proyecto, se define una nueva clase ListadoProyecto que contendrá una lista de proyectos y los métodos necesarios para gestionar esa lista. Por último se crea un programa principal (clase Main, con método main) con un menú que permitirá trabajar con la estructura de proyectos y trabajadores: introducir un trabajador, introducir un proyecto y asignar un trabajador a un proyecto.
